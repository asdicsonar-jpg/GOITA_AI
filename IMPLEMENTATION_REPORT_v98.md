## 1. 実施した変更(G/G_B 同一diff・4ハンク)

### レバー(a) ddKey の全単射コンパクト化(G: ddKey / G_B: ddKey)
各席8カウント(各0-8)をペアごとに base-9 で1バイトへ位置符号化(旧: 区切り`|`+8桁=**36文字** → **16文字**)。`ouV` も固定幅1バイト、終端の「開いたセル」は区切り不要。
```js
// 旧: for (const s of SEATS){ k += "|"; const c=counter(...); for(const t of KOMAS) k += (c[t]||0); }  // 4席×9=36文字
// 新:
for (const s of SEATS) {
  const c = counter(st.hands[s]);
  k += String.fromCharCode((c["王"]||0)*9+(c["飛"]||0), (c["角"]||0)*9+(c["金"]||0),
                           (c["銀"]||0)*9+(c["馬"]||0), (c["香"]||0)*9+(c["し"]||0));  // 4席×4=16文字
}
```
**単射性**: 全フィールドが固定幅の位置符号化(終端の可変長 open のみ末尾)であり、別の特徴組は必ず別キーになる。カウントは手駒≤8枚ゆえ各0-8で base-9 は無損失。→ オリジナルと**厳密に同一の等価類(衝突/ヒットパターン)**を保つ = 挙動不変。

### レバー(b) 決定化ごとの memo 即時解放(G/G_B: ddRootValues)
`ddRootValues` で `new Map()` を都度生成せず、単一プール `_ddMemoPool` を `clear()` して再利用。
```js
const _ddMemoPool = new Map();
function ddRootValues(st, seat, evalFn, actions, nodeCap) {
  const budget = {n: nodeCap || 300000};
  const memo = _ddMemoPool; memo.clear();   // 旧: const memo = new Map();
  ...
```
**等価性**: 各決定化は元来 memo 非共有(毎回 `new Map()`)であり、プールを空に `clear()` してから使うのは同一の初期条件 → 返り値は完全同一。`ddRootValues` は非再入(`ddSearch` の `evalFn` は `matchEquity`/`meqEval` のみで DD探索へ再帰しない・callerはmcDecideReceive等の逐次ループ)ゆえ単一プールで安全。同時生存する大規模 Map を**構造的に1本**へ制限し、促成モードでの複数世代 Map 滞留(v97 §5.2 のピークRSS要因)を抑える。
