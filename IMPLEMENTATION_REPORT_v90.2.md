## 0. サンドボックス再構築に関する注記

本セッション開始時、作業用サンドボックス（`/tmp`）が前セッションから引き継がれておらず、`goita-app-v90-fixed.zip`・`goita-app-v90b.zip`（byte-exact比較基準）からの再展開、npm依存関係（jsdom/axe-core）の再インストール、jsdomハーネス（`harness.js`）・実行時シム（`window.__T`公開用ブロック）の再構築を行いました。過去セッションで使っていたシムの完全な内容は保存されていなかったため、本コミットの検証に必要な範囲でシムを再設計しています。

**この再構築の過程で1件のハーネス側バグ（本体コードのバグではない）を発見・修正しました**: シム挿入位置を誤り、IIFE（`(() => { ... })();`）の**外側**にシムを挿入していたため `app`/`G` が参照できずテストが全滅していました。原因はIIFEの閉じ括弧の直前ではなく直後に挿入していたことで、挿入位置をIIFE内部（`})();`の直前）に修正して解決しました。本体コード（`index.html`）自体には影響していません。

## 1. 実施した変更

### F-3: し攻め宣言の枚数誤判定を修正

`auditOuShiSignal()`のshi_sig分岐で、宣言時保有枚数`shiN`の計算に「この局で自分が既に伏せたし」を加算するよう変更（計画のコード片をそのまま採用）。「し4枚→し伏せ→初手しで攻める」という伏せ同駒2倍狙いの正規戦略が、従来は攻め時点の手駒(3枚)だけを見て「薄い宣言」と誤判定されていた不具合を修正。honest判定・文言表示ともに新shiNを使用。王合図分岐（ou_sig）には一切触れていません。

### F-1: 親の初回伏せで推奨駒(.koma.rec)が飛び出ない不具合を修正

計画が特定した根本原因（`.koma.deal-in`の`dealIn`アニメが`.koma.rec`の`recGlow`をCSS詳細度同点・後方宣言優先で恒久上書き＝`fill-mode:both`により`.deal-in`クラス除去まで永続）どおり、**後片付け（クリーンアップ）方式**で実装:

- `dealAnimStart()`に、`app._dealAnimUntil`到達+250ms後に発火する単一ハンドルのクリーンアップタイマー（`app._dealCleanupT`）を追加。全`.koma.deal-in`要素から`deal-in`クラスと inline `animationDelay` を除去する`dealCleanup()`を新設。
- `#board`タップによるスキップハンドラにも同じ`dealCleanup()`を即時実行するよう追加。

計画が明示的に不採用とした代替案（推奨駒だけdeal-inを外す／CSS宣言順の入替）は採っていません。
